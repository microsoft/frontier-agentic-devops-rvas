## Task

Fix module-page "Learning paths" track cards that rendered as non-interactive `<div>` elements and did nothing on click.

## Root Cause

`renderTracks()` in `docs/assets/js/module.js` created plain `<div class="track-item">` with `cursor: pointer` styling but no `href` attribute or click handler — not focusable or keyboard-accessible.

## Solution Implemented

1. **Track cards → anchors:** Modified `renderTracks()` to render track items as `<a href="#track-${id}">` links when track has challenges (count > 0); keeps `<div>` for empty tracks (0 challenges) to avoid dead anchor links.

2. **Scroll targets:** Added `id="track-${trackId}"` to each `.group-head` div in `renderChallenges()`.

3. **Scroll spacing:** Added `scroll-margin-top: 72px` to `.group-head` in `docs/assets/css/styles.css` (58px navbar height + 14px breathing room).

4. **Smooth scroll:** Native `scroll-behavior: smooth` already on `html` element — anchor navigation works without custom JS.
