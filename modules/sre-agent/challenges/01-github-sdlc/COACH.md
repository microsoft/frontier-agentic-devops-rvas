# Coach Guide: Challenge 01 — Establish the GitHub SDLC

### Expected Outcome

A rough request becomes a clear issue, branch, pull request, review, and visible quality signal. The issue is also suitable as a first agent-ready specification.

### Strong Evidence

- Issue includes user impact, acceptance criteria, and validation.
- Issue includes boundaries, relevant files, non-goals, validation commands, and human checkpoints.
- Pull request references the issue.
- Review comments discuss risk or correctness.
- A check, test, or manual validation result is visible.

### Common Gaps

- Issue is only a title.
- Pull request includes unrelated changes.
- Review is reduced to a rubber-stamp approval.
- Validation is described verbally but not recorded.
- Agent-ready details are assumed from team conversation rather than written.

### Coach Hint

Ask: If a teammate joined tomorrow, could they reconstruct why this change happened and why it was safe?
