# Coach Guide: Challenge 00 — Setup and Team Launch

### Expected Outcome

Teams can access the repository, choose a development environment, run a baseline check or accept a coach fallback, and capture setup blockers visibly. They also name the human roles that stay accountable when agents assist: architect, reviewer, escalation handler, and operator.

### Validation Pattern

- Repository is open for every participant or pair.
- At least one team member can run the app, tests, or preflight.
- Blockers are written as issues or coach notes.
- The team understands that Challenges 1-4 are core and Challenges 5-6 may use guided simulations.
- Starter context artifact exists with roles, boundaries, decision location, and at least one review rule.

### Recovery Moves

- Move blocked participants into pairs.
- Provide a clean baseline branch.
- Skip local setup and use Codespaces if available.
- If neither environment path works, use coach screen-share for the first pull request while participants drive decisions.
- If teams start discussing governance only verbally, ask them to write the smallest useful context note before Challenge 01.
